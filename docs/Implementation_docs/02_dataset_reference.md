# ModelGuard — Meta-Dataset Reference

## File Location

```
data/meta_dataset.csv       ← full dataset (120 rows)
data/test_split.csv         ← held-out 20% test set (generated by Stage 2)
```

## Dataset Structure

- **Rows**: 120 — one per trained model
- **Columns**: 21 — 20 feature columns + 1 label column (`failure_mode`)
- **Label distribution**: 30 models per failure class (perfectly balanced)

---

## Feature Definitions

All features are **scalar values** derived from per-epoch training logs.
They are computed *after* training completes but use only information
available during training (no test set access).

### Raw Curve Finals

| Feature | Description |
|---|---|
| `final_train_loss` | Cross-entropy loss on training set at the last epoch |
| `final_val_loss` | Cross-entropy loss on validation set at the last epoch |
| `final_train_acc` | Classification accuracy on training set at the last epoch |
| `final_val_acc` | Classification accuracy on validation set at the last epoch |

### Gap Metrics (Generalisation Signals)

| Feature | Formula | What it signals |
|---|---|---|
| `loss_gap` | `final_val_loss − final_train_loss` | +ve and large → overfitting |
| `acc_gap` | `final_train_acc − final_val_acc` | +ve and large → overfitting |
| `overfit_score` | `max(0, acc_gap)` | Clipped overfitting signal (non-negative) |

### Trend Slopes (Trajectory Signals)

Computed via `numpy.polyfit` (linear regression) over the **last 10 epochs**
(or the full curve if fewer than 10 epochs were recorded).

| Feature | What it signals |
|---|---|
| `val_loss_trend` | Positive → val loss still rising at end of training (bad) |
| `train_loss_trend` | Negative → training still converging (normal) |

### Volatility (Stability Signals)

Standard deviation across all epochs:

| Feature | What it signals |
|---|---|
| `loss_volatility` | High → val loss is erratic; may indicate label noise or instability |
| `acc_volatility` | High → val accuracy is erratic across epochs |

### Best Performance

| Feature | What it signals |
|---|---|
| `best_val_acc` | Peak validation accuracy achieved at any epoch |
| `best_val_epoch` | Epoch index (0-based) at which peak val acc occurred; early peak → diverging after |

### Convergence

| Feature | Value | What it signals |
|---|---|---|
| `convergence_speed` | Epoch index where `train_loss` first dropped below 0.5 | `-1` if never reached — indicates failure to learn |

### Curve Correlation

| Feature | What it signals |
|---|---|
| `train_val_loss_corr` | Pearson r between train_loss and val_loss curves. High positive → both falling together (healthy). Near zero or negative → curves decoupled (overfit or noise) |

### Gradient Norm Statistics

Computed as the mean L2 norm of all parameter gradients, averaged across batches, per epoch.

| Feature | What it signals |
|---|---|
| `gradient_norm_mean` | Low → vanishing gradients; Very high → exploding gradients |
| `gradient_norm_std` | High → training is unstable, gradients fluctuating heavily |

### Dataset & Architecture Metadata

| Feature | Values | What it signals |
|---|---|---|
| `class_entropy` | Float (nats) | Low → class imbalance; High → balanced distribution |
| `dataset_source` | 0, 1, or 2 | 0 = sklearn tabular, 1 = FashionMNIST, 2 = CIFAR-10 |
| `num_params` | Integer | Total trainable parameters; large → higher overfit risk |

---

## Label Column

| Value | Name | Description |
|---|---|---|
| 0 | `OVERFIT` | Model over-parameterised, no regularisation |
| 1 | `CLASS_IMBALANCE` | Training set heavily skewed toward 2 dominant classes |
| 2 | `LABEL_NOISE` | 30% of training labels randomly flipped |
| 3 | `HEALTHY` | Balanced data, dropout regularisation, clean labels |

---

## Failure Injection Parameters (Reproducible)

All experiments are seeded with `random.seed(42)`, `numpy.random.seed(42)`,
`torch.manual_seed(42)`, `torch.cuda.manual_seed_all(42)`.

| Class | Architecture | Dropout | Epochs | Special Transform |
|---|---|---|---|---|
| OVERFIT | [512, 512, 512, 512] | 0.0 | 50 | None |
| CLASS_IMBALANCE | [128, 64] | 0.0 | 30 | 90% from 2 classes, 10% from rest |
| LABEL_NOISE | [128, 64] | 0.0 | 30 | 30% label flip to random wrong class |
| HEALTHY | [128, 128] | 0.3 | 30 | None (balanced data) |

Training config (all models): Adam optimiser, lr=0.001, CrossEntropyLoss, batch size 256 (GPU) / 64 (CPU).

## Dataset Distribution per Class × Source

Each failure class cycles through the 3 datasets across its 30 runs:

| Run index (within class) | Dataset used |
|---|---|
| 0, 3, 6, 9, … | sklearn tabular (source=0) |
| 1, 4, 7, 10, … | FashionMNIST (source=1) |
| 2, 5, 8, 11, … | CIFAR-10 (source=2) |

This means each failure mode is represented across all three data domains,
making the meta-classifier generalisable across data types.
